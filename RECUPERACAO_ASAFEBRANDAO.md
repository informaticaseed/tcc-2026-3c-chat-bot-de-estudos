1. O que é uma Issue e para que serve no desenvolvimento de software?
Uma Issue é uma tarefa, problema ou sugestão registrada dentro de um projeto de software. Ela serve para organizar o trabalho da equipe, deixando claro o que precisa ser corrigido, criado ou melhorado. Assim, os desenvolvedores conseguem acompanhar as atividades e dividir melhor as responsabilidades.

2. O que é um Pull Request e por que ele é importante?
Um Pull Request é uma solicitação para juntar alterações feitas em uma parte do projeto com o código principal. Ele é importante porque permite que outras pessoas revisem o código antes de ele ser aplicado, evitando erros e melhorando a qualidade do sistema.

3. O que é separação de camadas no código? Explique a diferença entre app.py e repositorio.py e por que isso importa.
Separação de camadas é uma organização do código onde cada arquivo ou parte do sistema tem uma função específica. O app.py fica responsável pela aplicação, como receber requisições, controlar páginas e chamar as funções necessárias.
Já o repositorio.py fica responsável pelo acesso ao banco de dados, como inserir, buscar, alterar e excluir informações. Isso importa porque deixa o código mais organizado, fácil de entender e de manter.
